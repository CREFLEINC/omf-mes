OMF Spec · ISA-95 Part 3 MOM

MOM 기능 정의서
8개 활동 × 기능 · AI 적용 · OMF 우선순위

ISA-95 Part 3 제조운영관리(MOM)의 8개 활동을 기준으로, 각 활동의 정의·주요 기능·입출력·MES 제품 대응·AI Agent 적용 가능성·OMF 구현 우선순위를 정리한 기능 설계 기준 문서.

본 문서는 생산(Production) 범주를 기준으로 8개 활동을 정의한다. ISA-95는 동일한 8개 활동 틀을 품질·설비/보전·재고 범주에도 그대로 적용하므로(범주명만 교체), 아래 정의는 4대 범주에 공통으로 확장된다.

구조 — 표준상 8개 활동은 동등하나, 정보 흐름상 지원 활동 2개(제품 정의 관리 · 자원 관리)가 실행 루프 ①~⑥(스케줄링 → 디스패칭 → 실행 → 수집 → 추적 → 분석)에 기준 정보를 공급한다(편의적 그룹핑).

AI 보조 ML·규칙 기반(분석·추천) AI 코파일럿 생성형·자연어(NLQ) AI 자율 agentic(자동 결정·실행) OMF v1 MES 코어 OMF v2 MOM 확장
지원 활동실행 루프에 기준 정보 공급 (상시)
정의

제품 정의 관리

Product Definition Management (일반화: Operations Definition)
AI 보조v1

무엇을 어떻게 만들지에 대한 기준 정보(마스터 레시피·BOM·라우팅·작업표준)를 생성·버전관리·배포한다.

checklist주요 기능
  • 제품 정의·레시피 관리
  • BOM·공정 라우팅 관리
  • 작업표준·규격·문서
  • 버전·변경이력(ECN) 관리·배포
sync_alt입력 · 출력 / 대응
입력제품 설계·공정 엔지니어링 정보
출력실행에 쓰일 제품 정의(레시피)
MES 대응: 레시피관리(미라콤 RMS), 사양·문서 관리 · MESA-11: Document Control
AI: 변경 영향 분석, 레시피 파라미터 추천(보조)
자원

자원 관리

Production Resource Management (일반화: Resource Management)
AI 보조v1

인원·설비·자재·치공구의 가용성·능력(capability)·상태·예약을 관리해 스케줄링에 공급한다.

checklist주요 기능
  • 자원 상태·가용성 관리
  • 능력(스킬·인증·교정) 관리
  • 예약·할당
  • 설비 적격성(qualification)·교정 주기 관리
sync_alt입력 · 출력 / 대응
입력자원 마스터 + 실시간 상태
출력가용성·능력 정보(availability·capability) → 스케줄링
MES 대응: 설비관리(RAS)·인력관리·자재(INV) 일부 · MESA-11: Resource Allocation, Labor Mgmt
AI: 자원 가용성·소요 예측(보조)
실행 루프 ①~⑥계획 → 실행 → 측정 → 피드백
①

상세 생산 스케줄링

Detailed Production Scheduling
AI 자율(글로벌 일부)v1v2 최적화

L4의 거친 생산계획을 현장 자원·제약에 맞춰 분 단위로 순서화·최적화한다.

checklist주요 기능
  • 작업 순서화·시퀀싱
  • 자원·제약 반영, 셋업 최소화
  • 납기 기반 우선순위
  • 변동 발생 시 재계획(re-scheduling)
sync_alt입력 · 출력 / 대응
입력생산계획(L4) + 자원 가용
출력상세 일정(작업 순서)
MES 대응: 삼성SDS Schedule&Dispatch / 전문 APS는 VMS MOZART · MESA-11: Operations Scheduling
AI: 자동 재계획·최적화(APS)는 성숙. 자율 에이전트는 SAP Joule(생산계획·운영, 2026 Q1 GA 예정) 단일 사례로 초기. OMF: v1=기본 순서화·재계획, v2=자율 최적화 에이전트
②

생산 디스패칭

Production Dispatching
AI 보조v1

상세 일정에서 도출된 작업을 설비·작업자에게 배정·릴리스(디스패치)한다("작업 하달"). 릴리스된 작업은 ③ 실행 관리에 Job Order로 전달된다.

checklist주요 기능
  • 작업 배정·작업지시 발행
  • 실시간 우선순위 조정
  • WIP 흐름 통제
  • 디스패치 리스트 관리
sync_alt입력 · 출력 / 대응
입력상세 일정
출력디스패치 리스트(릴리스된 작업) → 실행
MES 대응: Schedule&Dispatch, 생산관리(WIP) · MESA-11: Dispatching Production Units
AI: 실시간 우선순위 재배정(보조~부분 자율)
③

생산 실행 관리

Production Execution Management
AI 보조(보수적)v1

하달된 작업(Job Order)이 규격대로 실제 수행되도록 지휘·통제한다. ②(배정·하달)와 ④(수집·기록) 사이의 '실행 통제' 계층.

checklist주요 기능
  • 작업 수행 지시·레시피 적용
  • 설비·인원 명령 조정(L2 연동)
  • 자원 확인·투입, 공정 순서 보장
  • 작업 상태 관리·실행 중 예외 처리(인터록·포카요케)
sync_alt입력 · 출력 / 대응
입력Job Order
출력실행 결과 → 데이터 수집
MES 대응: EWI(전자작업지시)·레시피 다운로드·Track-in/out·인터록·설비제어 연동 · MESA-11: Process Management
AI: 예외·대체 라우팅 제안 — 안전·품질 직결이라 "제안 후 승인" 권장(자율은 보수적)
④

생산 데이터 수집

Production Data Collection
AI 부분자율(FDC)v1

실행 중 발생한 실제 데이터·이벤트(설비·작업자·검사장비)를 수집·적재한다. (품질 검사 결과는 표준상 품질 범주에 1차 귀속되나, CREFLE 비전검사 수직통합 시 이 단계에서 함께 수집된다.)

checklist주요 기능
  • 설비 데이터 수집(인터페이스)
  • 수동/자동 입력
  • 이벤트·알람 수집
  • 품질 측정값·검사 결과 수집
sync_alt입력 · 출력 / 대응
입력현장 L1/L2·검사장비
출력수집 데이터 → 추적·분석
MES 대응: 설비연동(삼성SDS Machine Control/EE)·실시간 모니터링(LG CNS)·FDC/EPT · MESA-11: Data Collection
AI: 스트림 이상 자동 감지(FDC) — 부분 자율
⑤

생산 추적

Production Tracking
AI 코파일럿(NLQ)v1

진척·실적을 집계하고 로트·시리얼 이력(genealogy)을 구성하며, L4 보고용 실적(Performance)을 준비한다.

checklist주요 기능
  • WIP 추적
  • 정/역 추적성(genealogy)
  • 실적 집계·L4 보고
  • 라벨·시리얼 관리
sync_alt입력 · 출력 / 대응
입력수집 데이터
출력생산 실적·이력
MES 대응: 생산관리(WIP)·라벨(LG CNS)·추적성(CREFLE Flowtrack) · MESA-11: Product Tracking & Genealogy
AI: 자연어로 이력 질의("이 로트 어디 거쳤어?") — 코파일럿
⑥

생산 성과 분석

Production Performance Analysis
AI 코파일럿/보조v2 차별점

KPI·수율·OEE·사이클타임을 분석해 개선 피드백을 만든다. 한국 벤더는 기본 리포트는 본체에 두되 심층 분석·예측(AI) 계층을 별도 제품으로 분리 — AI Agent 화이트 스페이스의 핵심 지점.

checklist주요 기능
  • OEE·수율·FPY·스크랩 분석
  • 병목·손실 분석
  • 근본원인 분석(RCA)
  • 리포트·대시보드, 개선 피드백
sync_alt입력 · 출력 / 대응
입력실적·이력·품질 데이터
출력인사이트 → 스케줄링·정의 피드백
MES 대응: ※심층 분석은 대부분 별도 제품 — Brightics·Nexplant Analytics·BISTel eDataLyzer(ML 분석 플랫폼) · MESA-11: Performance Analysis
AI: 품질예측·RCA·NLQ — OMF 차별화의 핵심(글로벌 선도 벤더 도입 중: 생성형·LLM+RAG, 자율은 초기 / 한국 본체엔 부재)

OMF 적용 요약 — 실행 루프 ①~⑤ + 지원 2개는 v1 MES 코어로 먼저 구축(시장 진입), ⑥ 성과 분석과 ①의 고도 최적화는 v2 AI Agent 확장으로 차별화. 특히 ④ 데이터 수집 ↔ ⑥ 분석 구간에 CREFLE 비전검사를 수직 통합(검사 결과를 ④→⑥로 직결)하면, 별도 QMS 도입 부담을 낮추면서 품질 분석을 강화할 수 있다.

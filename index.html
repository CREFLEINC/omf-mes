한국 벤더는 AI/ML 기능을 MES 본체가 아니라 별도의 분석·엔지니어링 제품으로 분리해 제공하는 패턴을 보인다 — 삼성SDS의 Brightics·Nexplant Analytics, 미라콤 EES의 FDC/EPT/sDM, VMS의 MOZART 제조AI, BISTel eDataLyzer 등. 이들 대부분은 ML 기반 분석·예측 수준이며, 한 화면·한 대화창에서 작동하는 통합형 대화·자율 에이전트 경험은 아직 약하다.

01

MES 기능 표준 프레임 (ISA-95)

AI Agent가 얹히는 계층의 정의

ISA-95 / IEC 62264 자동화 계층은 Level 0~4의 5단계로 구성된다. MES(제조실행시스템 = 제조운영관리, MOM)는 Level 3에 위치하며, "원하는 최종 제품을 생산하기 위한 워크플로우 활동을 정의"하는 계층이다.

즉 MES에 AI Agent를 적용한다는 것은 결국 이 Level 3 워크플로우 — 생산지시·실적·품질·설비·자재 — 위에 지능형 에이전트를 얹는다는 의미다. 본 리포트의 기능 분석은 이 표준 프레임을 기준으로 한다.

한국 vs 글로벌 — 핵심 격차

제품 단위 AI Agent 통합 수준

factory한국 벤더

  • ML 기반 분석·예측은 강함 — FDC 이상탐지, 설비 성능 추적, RCA, APS 최적화
  • 그러나 LLM 기반 대화형·자율 에이전트는 MES 본 제품에 거의 부재
  • AI가 별도 제품으로 분리 — MES와 AI가 한 경험으로 통합되지 않음

smart_toy글로벌 벤더

  • 명시적 'AI Agent / Copilot / 자율 에이전트'를 MES·산업 플랫폼 안에 탑재
  • SAP의 자율 주문 검증·릴리스, Siemens·AVEVA의 자연어 인터페이스
  • 단, 다수가 GA 예정/베타/시연 단계 — 실증 제품은 아직 제한적

06

신뢰성 주의사항

해석 시 반드시 함께 고려
  1. 시간 민감성 — SAP 생산 에이전트는 2026 Q1 GA 예정, Rockwell-NVIDIA는 시연 예정, AVEVA MES 질의는 Open Beta. 모두 "실증된 GA 제품"이 아니라 발표/로드맵 단계.
  2. 출처 품질 — 한국 벤더 정보는 대부분 벤더 자체 마케팅 페이지·데이터시트 의존. 기능 존재 여부엔 적절하나 성능 수치(예: 정비시간 25%↓)는 독립 검증 안 됨(벤더 주장).
  3. '본 제품' vs '확장/별도 제품' 경계 — 삼성SDS MES 본 제품엔 AI 없지만 Brightics 등 별도 제품엔 있음. 미라콤 FDC/EPT/sDM은 MES Core가 아닌 EES 확장. "MES에 AI가 있다/없다"는 어느 제품 단위로 보느냐에 따라 달라진다.
  4. 반박된 주장 주의 — BISTel RCA·VMS 제조AI를 '자율/agentic'으로 규정 / LG CNS·미라콤에 'AI 언급 전무' 주장은 검증에서 반박됨(0-3 또는 1-2). 단정하지 말 것.
  5. 삼성SDS 한국어 본 제품 URL이 현재 /error 리다이렉트 → US/EN 미러로 검증.
